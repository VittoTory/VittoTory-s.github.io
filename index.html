8. Исключения (exceptions)

| Что может пойти не так?

  • При реализации шага №1 (гитхаб как хостинг) возникла проблема у одного из участников проекта — во время работы над одним из проектов предыдущего семестра, уже был создан хостинг — репозиторий формата username.github.io. В связи с этим отчёт всего один. Возможно далее это будет исправлено.
  • Также стоит отметить, что исключением для системы будет являться то, что скорее всего придётся использовать внешний хостинг, так как бесплатная версия GitHub не позволяет использовать БД. 
  • Проектируемая система должна использоваться только неспосредственными проектировщиками БЛВС, то есть будет иметь ограниченный доступ. 

9. Альтернативы (alternates)

| Что может повлиять на путь перехода от предусловий к постусловиям?

  • Для ограничения доступа к системе можно создать модуль авторизации. Но оптимальным и разумным решением будет ограничение доступа на сетевом уровне с помощью Access-листов.
